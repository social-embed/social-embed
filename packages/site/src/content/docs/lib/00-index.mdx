`@social-embed/lib` is a zero-dependency JavaScript library for matching media URLs, extracting structured IDs, and generating embeddable URLs. It runs in any JavaScript environment — browsers, Node.js, edge workers.

```ts
import { MatcherRegistry } from "@social-embed/lib";

const registry = MatcherRegistry.withDefaults();
registry.toEmbedUrl("https://youtu.be/Bd8_vO5zrjo");
// "https://www.youtube-nocookie.com/embed/Bd8_vO5zrjo" (privacy-enhanced by default)
```

## Key features

- **Unified matching** — `registry.match()` and `registry.toEmbedUrl()` across all built-in platforms
- **Type-safe matchers** — `UrlMatcher` + `defineIframeMatcher` / `defineScriptMatcher` for custom services
- **SSR-friendly core** — Immutable `MatcherRegistry`; optional `RegistryStore` and `@social-embed/lib/browser` for reactive / CDN usage
- **Zero dependencies** — No npm dependencies, no DOM requirement in the core entry
- **Migration path** — Deprecated `convertUrlToEmbedUrl()` still works; new code should use the registry API

## How it fits

This library handles steps 1-3 of the social-embed pipeline (detect → extract → build). It is used internally by the [`<o-embed>` web component](/wc/), but you can use it standalone for:

- **Server-side URL validation** — `registry.match()` before storing user-submitted URLs
- **Custom renderers** — Build your own embed UI in React, Vue, Svelte, or any framework
- **Content pipelines** — Transform URLs in batch processing, API endpoints, or build steps
